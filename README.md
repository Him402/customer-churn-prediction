# Customer Churn Prediction

A machine learning project that predicts whether a telecom customer is likely to churn, built as a portfolio data science project. The project covers the full ML lifecycle: data cleaning, feature engineering, model selection via GridSearchCV, business-driven threshold optimisation using a profit curve, SHAP-based explainability, and a deployed Streamlit web application.

---

## Problem Statement

Customer churn is one of the most costly problems in subscription-based businesses. Acquiring a new customer typically costs five to ten times more than retaining an existing one. This project builds a binary classifier that identifies high-risk customers before they leave, enabling the business to target them with retention offers. The model also estimates the optimal intervention threshold by maximising expected annual profit — not just classification accuracy.

---

## Dataset

This project uses the **IBM Telco Customer Churn** dataset, which is publicly available on Kaggle.

**Download link:** https://www.kaggle.com/datasets/blastchar/telco-customer-churn

Once downloaded, place the file in the root of this repository with the exact filename:

```
WA_Fn-UseC_-Telco-Customer-Churn.csv
```

The dataset contains 7,043 customer records with 21 features including tenure, contract type, internet service, monthly charges, and a binary churn label.

## Methodology

**Feature Selection.** Eight features were selected based on domain knowledge: tenure, monthly charges, total charges, contract type, internet service, online security, tech support, and payment method.

**Preprocessing.** Numeric features are standardised using `StandardScaler`. Categorical features are encoded using `OneHotEncoder`. Both transformers are bundled with the classifier inside a single `sklearn Pipeline`, which prevents data leakage by ensuring the scaler is fitted only on training data.

**Model Selection.** Two models were evaluated using 5-fold cross-validated GridSearchCV on training data only: Logistic Regression (interpretable baseline) and Random Forest (non-linear). The model with the higher ROC-AUC on a held-out validation set was selected.

**Threshold Optimisation.** Rather than using the default 0.5 decision threshold, a profit curve was computed on validation data using realistic business assumptions (average monthly revenue per customer and cost of a retention offer). The threshold that maximised expected annual profit was chosen as the deployment threshold.

**Explainability.** SHAP (SHapley Additive exPlanations) values are computed at prediction time to show which features drove each individual prediction. `TreeExplainer` is used for Random Forest and `LinearExplainer` for Logistic Regression.

## Setup and Installation

**Step 1.** Clone this repository.

```bash
git clone https://github.com/Him402/customer-churn-prediction.git
cd customer-churn-prediction
```

**Step 2.** Create and activate a virtual environment.

```bash
python -m venv churn_env

# Windows
churn_env\Scripts\activate

# Mac / Linux
source churn_env/bin/activate
```

**Step 3.** Install dependencies.

```bash
pip install -r requirements.txt
```

**Step 4.** Download the dataset from Kaggle (link above) and place it in the project root as `WA_Fn-UseC_-Telco-Customer-Churn.csv`.

**Step 5.** Run the training notebook end to end to generate the model file.

Open `Churn_model_GridSearch_Validation.ipynb` in Jupyter, make sure the kernel is set to your `churn_env` environment, and run all cells from top to bottom. This will save `churn_best_pipeline.pkl` in the project root.

```bash
pip install ipykernel
python -m ipykernel install --user --name=churn_env --display-name "Churn Project"
jupyter notebook
```

**Step 6.** Launch the Streamlit app from the same activated environment.

```bash
streamlit run app.py
```

The app will open automatically in your browser at `http://localhost:8501`.

## Streamlit App Features

The web app allows a user to enter a customer's details through an interactive form and receive an instant churn prediction. Specifically it shows the predicted churn probability, a risk verdict with a recommended retention action, a SHAP waterfall chart explaining which features drove that individual prediction, a bar chart of the top 10 most impactful features, and an adjustable decision threshold slider (defaulting to the profit-optimised value from training).
